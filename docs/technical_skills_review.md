# Technical Skills for Implementation Plan

Based on the `CATALOG.md`, the following skills are best suited for creating a high-fidelity, production-grade implementation plan for the **Expense Approval Portal**:

## Architecture & Design
- **`architect-review`**: For senior-level structural validation of the proposed system.
- **`senior-architect`**: To ensure scalability and maintainability across the React/Node stack.
- **`architecture-decision-records`**: To document the "why" behind key technical choices.

## Development Standards
- **`frontend-dev-guidelines`**: Opinionated standards for the React/TypeScript frontend.
- **`backend-dev-guidelines`**: Standards for the Node.js/Express/TypeScript API.
- **`nodejs-backend-patterns`**: Best practices for middleware, auth, and database integration.

## Quality & Compliance
- **`production-code-audit`**: To ensure the implementation meets corporate-level standards.
- **`testing-patterns`**: For a robust TDD approach with Jest and mocking.
- **`security-auditor`**: To proactively address vulnerabilities in the financial portal.
- **`wcag-audit-patterns`**: To ensure accessibility compliance (WCAG 2.2).

## Methodology
- **`brainstorming`**: (Active) To transform the prompt into a validated design.
- **`code-refactoring-refactor-clean`**: To enforce SOLID principles and clean code.
